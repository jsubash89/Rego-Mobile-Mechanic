# ReGo Mobile Mechanic Linear Tracking Plan

Linear project: ReGo Mobile Mechanic
Team: JS Ventures

Tracking policy:
- Each major feature document gets a Linear issue under this project.
- The issue title mirrors the feature document title.
- The issue description links to the repo doc path and summarizes the build contract.
- Implementation work should reference the feature doc and update it when behavior changes.
- A feature cannot be considered done until its doc, acceptance criteria, tests, and implementation agree.

Initial epics/workstreams:
1. Documentation control plane
2. Customer booking MVP
3. Provider operations MVP
4. Dispatch/admin MVP
5. Platform foundations
6. Payments, notifications, support, and trust
